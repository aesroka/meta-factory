# SPIN Selling — Framework Summary

## Core Principle

High-value sales require consultative questioning that helps buyers **discover their own needs**. The seller guides, but the buyer convinces themselves.

## The SPIN Sequence

### S — Situation Questions

**Purpose**: Establish context and baseline understanding.

**Examples**:
- "How many people are involved in this process?"
- "What system do you currently use for X?"
- "How long have you been doing it this way?"
- "Who is responsible for this area?"

**Rules**:
- Keep these minimal — they're boring for the buyer
- Do your homework beforehand to reduce situation questions
- Never ask what you could have Googled

### P — Problem Questions

**Purpose**: Surface difficulties, dissatisfactions, and pain points.

**Examples**:
- "What challenges do you face with the current approach?"
- "Are you satisfied with the reliability of...?"
- "How difficult is it to...?"
- "Where do things break down?"
- "What's frustrating about the current process?"

**Rules**:
- Most salespeople stop here — don't
- Problems alone don't create urgency to buy
- You need to develop the implications

### I — Implication Questions

**Purpose**: Develop the problem's consequences. Make the pain tangible.

**Examples**:
- "What effect does that have on throughput?"
- "How does that impact your team's morale?"
- "What does that mean for your ability to hit targets?"
- "If this continues, what happens to...?"
- "How much time do you lose when that happens?"

**Rules**:
- This is where value is created
- Connect problems to business outcomes
- Quantify wherever possible
- Build urgency through logical consequences

### N — Need-Payoff Questions

**Purpose**: Get the buyer to articulate the value of solving the problem.

**Examples**:
- "How would it help if you could...?"
- "What would it mean to your team if this were solved?"
- "Why is it important to resolve this?"
- "What benefits would you see from...?"
- "If we could address X, how would that affect Y?"

**Rules**:
- Let the buyer sell themselves
- They state the benefits, not you
- Creates psychological ownership of the solution

## The Implication-Payoff Balance

| Seller Focus | Buyer Perception |
|-------------|------------------|
| All Problems, no Implications | "Interesting but not urgent" |
| All Implications, no Need-Payoff | "Doom and gloom salesperson" |
| Strong Implications + Need-Payoff | "I need to solve this now" |

## Quantifying Pain (Integration with Mom Test)

For each problem identified, develop:

1. **Frequency**: "How often does this happen?"
2. **Duration**: "How long does it take to fix?"
3. **Impact**: "What's the knock-on effect?"
4. **Cost**: "What does this cost in hard numbers?"

## Building the Business Case

Use Implication questions to build the equation:

```
Problem Frequency × Cost per Incident = Annual Cost of Problem
Annual Cost of Problem > Cost of Solution = Compelling ROI
```

## Question Flow Map

```
SITUATION → Sets context
     ↓
PROBLEM → Surfaces pain
     ↓
IMPLICATION → Develops consequences (creates urgency)
     ↓
NEED-PAYOFF → Buyer states value (creates ownership)
```

## Anti-Patterns to Avoid

- **Feature dumping**: Presenting solutions before developing needs
- **Premature closing**: Asking for commitment before value is established
- **Problem hopping**: Jumping between problems without developing any
- **Implication skipping**: Going straight from problem to solution
